NexClinical — Senior Dev One‑Pager

Last Updated: Jan 17, 2026 • Environment: Production

Next.js + Sanity • Vercel • Resend

Architecture

  • Frontend: Next.js 15 (App Router), React 19, TypeScript
  • UI: Tailwind CSS, shadcn/ui
  • CMS: Sanity Studio + GROQ + Image CDN
  • Rendering: PageBuilder maps block _type → component
  • Deployment: Vercel (ISR 60s, edge cache)
  • Email: Resend API for contact form

Data Flow (5 lines)

  • Edit in Sanity Studio → publish to production
  • Next.js server components fetch via GROQ
  • ISR cache revalidates every ~60s
  • Images resolved via Sanity CDN
  • No redeploy → content goes live within revalidation window

Key Decisions & Trade‑offs

  • Sanity vs self‑hosted: Hosted, fast, CDN → vendor dependency
  • ISR vs SSR only: Faster + cached → short staleness (≤60s)
  • Blocks/Page Builder: Non‑dev editing → needs schema rigor
  • GROQ vs GraphQL: Native/simple → smaller ecosystem
  • Resend vs SMTP: Reliable insights → cost vs DIY

Live Demo Checklist

  • Create a Page with Hero + Services blocks → visit /[slug]
